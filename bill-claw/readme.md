

# OpenClaw Skill 生成提示词（优化版）

你的任务是：**设计并实现一个可用于 OpenClaw 的记账 Skill（生产级别）**。

该 Skill 需要具备完整的**记账、分类管理、数据分析、可视化展示能力**，并能够通过自然语言与用户交互。

---

# 一、交付产物

你必须输出以下内容：

## 1. `SKILL.md`

用于指导 AI Agent 如何使用该 Skill，必须包含：

* Skill 功能说明
* 支持的用户意图（Intent）列表
* 每个 Intent 对应的处理流程
* 如何调用脚本（输入 / 输出规范）
* 示例对话（覆盖常见场景）
* 异常处理策略

## 2. Python 脚本

建议拆分为多个文件（结构清晰），置于 `scripts/` 目录：

* `scripts/main.py`：入口（解析指令 / 调度）
* `scripts/parser.py`：自然语言解析（时间 / 金额 / 类别）
* `scripts/db.py`：SQLite 数据库操作
* `scripts/report.py`：报表生成
* `scripts/web.py`：Web 看板服务
* `scripts/utils.py`：通用工具函数

---

# 二、技术约束

必须满足：

* 使用 **Python 3**
* 使用 **SQLite** 作为唯一数据库
* 使用 **虚拟环境（venv）**
* 遵循**最少依赖原则**（优先标准库，可用轻量库如：`dateparser`、`pydantic`、`matplotlib`）
* 项目应可直接本地运行

---

# 三、核心能力设计

## 1. 自然语言记账（核心能力）

系统需要从用户输入中提取结构化数据：

### 必须字段：

* 时间（time）
* 金额（amount）
* 类型（收入 / 支出）
* 类别（category）
* 备注（note）

---

### 1.1 时间解析

支持：

* 绝对时间：`2026-03-20`
* 相对时间：

  * 今天 / 昨天 / 前天
  * 上周一 / 上个月
  * 刚刚 / 晚上 / 中午

要求：

* 基于当前时间推导具体 datetime
* 若解析不确定 → 主动询问用户确认

---

### 1.2 金额解析

必须支持：

* 数字：`12.5`
* 中文：`十二块五`
* 简写：`23w`（= 230000）
* 混合表达：`1k`, `3千`, `2.3万`

要求：

* 统一转换为 float（单位：元）
* 具备**异常检测能力**：

  * 如：「吃饭花了 10000」
  * 应主动询问用户是否输入错误，而不是直接记录

---

### 1.3 分类系统

#### 默认分类：

**支出：**

* 正餐、零食饮料、出行、购物、日常开销、娱乐、居住、医疗健康、家人、社交、其他

**收入：**

* 工资、奖金、投资、家人、其他收入

---

#### 分类规则：

* 自动根据语义匹配分类：

  * 吃饭 → 正餐
  * 打车 → 出行
* 未匹配 → 归类为「其他」

---

#### 分类演进能力（重要）

系统需要支持：

1. **新增分类**

   * 示例：`帮我加一个“恋爱”分类`

2. **分类归并（带确认）**

   * 示例：

     ```
     把之前和约会相关的消费归到“恋爱”
     ```
   * 必须：

     * 先列出候选记录
     * 用户确认后再修改

---

### 1.4 删除记录

支持自然语言删除：

* 示例：

  * 删除昨天吃饭的记录
  * 删除上周所有打车

要求：

* 必须先展示将被删除的数据
* 用户确认后执行删除

---

## 2. 数据报表能力

根据时间范围生成统计分析：

---

### 2.1 基础统计

必须包含：

1. 总收入
2. 总支出
3. 净结余

---

### 2.2 分类统计

* 按类别汇总
* 输出占比

---

### 2.3 输出形式

* **数据部分 → 图片（图表）**
* **总结部分 → 自然语言**

---

### 2.4 总结风格（重要）

要求：

* 不要机械描述
* 更像朋友 / 恋人语气
* 可以轻微幽默

示例：

> “这个月你在奶茶上的投入已经可以开一家小店了，要不要考虑我入股？😏”

---

## 3. Web 看板

支持本地 Web 可视化：

---

### 触发方式

用户说：

* 开启 web 看板
* 打开账本页面

---

### 行为：

* 启动本地 HTTP 服务
* 返回访问地址（如：[http://localhost:8000）](http://localhost:8000）)

---

### 页面内容：

必须包含：

1. 记账记录列表
2. 分类统计图
3. 收支趋势图

---

### 技术建议：

* 使用：

  * `Flask`（轻量）
  * 或 `http.server` + 简单前端

---

# 四、Agent 行为规范（关键）

## 1. 不要盲目执行

在以下情况必须询问用户：

* 金额异常
* 分类不明确
* 删除 / 修改操作

---

## 2. 所有“修改类操作”必须二次确认

包括：

* 删除
* 分类迁移
* 批量修改

---

## 3. 输出风格

* 简洁 + 有温度
* 不要系统提示感太强

---

# 五、示例交互（必须写入 SKILL.md）

至少覆盖：

1. 正常记账
2. 模糊时间
3. 金额异常
4. 分类新增
5. 删除记录
6. 报表生成
7. Web 看板启动

## 实现与运行

本仓库已实现 CLI + SQLite + 报表与 Web 看板。实现代码在 `scripts/` 目录。安装与命令示例见 [SETUP.md](SETUP.md)；Agent 使用说明见 [SKILL.md](SKILL.md)。

在项目根目录可执行：`python scripts/main.py <子命令> ...`；也可使用根目录转发入口：`python main.py <子命令> ...`（等价）。
